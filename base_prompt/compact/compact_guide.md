# 对话压缩任务

你正在执行对话历史压缩任务。下方是待压缩的对话记录。

# 压缩要求

1. **提取关键信息**：保留对话中的重要事实、用户需求、已完成的步骤和待办事项。
2. **去除冗余**：省略闲聊、重复确认、无意义的来回。
3. **保持上下文连贯**：压缩后的摘要应能让后续对话无缝衔接，不丢失关键背景。
4. **输出格式**：只输出一段精炼的摘要文本，不要包含任何额外解释或元数据，不要使用"以下是压缩结果"之类的前缀。

# 注意事项

- 如果对话中包含工具调用及其结果，需保留工具执行的关键结论。
- 如果用户表达了明确的偏好或约束，务必在摘要中保留。
- 摘要长度应与原始对话的信息密度匹配：信息量大的对话应有更详细的摘要。
